# 品質評分機制

## 科技新聞評分公式

每則科技新聞的品質分數（quality_score）計算方式：

### 基礎分（Base Score）
- 所有新聞起始分 = **5**

### 加分項
| 條件 | 分數 | 說明 |
|------|------|------|
| 優先來源 (priority: true) | +3 | 在 TECH_SOURCES.json 中標記為 priority 的來源 |
| 多來源交叉報導 | +5 | 同一事件被 2+ 不同來源報導 |
| 時效性（24h 內）| +2 | 過去 24 小時內發布 |
| 高互動（Reddit score > 500）| +3 | Reddit 高分帖子 |
| 中互動（Reddit score > 200）| +1 | Reddit 中等分帖子 |
| GitHub 高星（daily stars > 50）| +3 | GitHub Trending 熱門項目 |

### 減分項
| 條件 | 分數 | 說明 |
|------|------|------|
| 昨日已報導 | -5 | 與前一天摘要重複的故事 |
| 低品質來源 | -2 | 來源不在 TECH_SOURCES.json 中 |

### 門檻
- **最低發布分 = 5** — 低於 5 分的新聞不納入摘要
- 每個 Topic 按 quality_score 降序排列

## 時事新聞評分

時事新聞的品質評估不使用數字分數，而是由 AI 根據以下優先級判斷：

1. **重大突發事件** — 影響多國、涉及衝突/災難/重大政策
2. **多國媒體報導** — 越多國家報導越重要
3. **對讀者直接影響** — 尤其是對日本/台灣/東亞的影響
4. **時效性** — 今日發生 > 持續發展 > 背景分析
5. **獨特性** — 與昨日不重複

## 去重規則

### 科技新聞去重
- 標題相似度 > 85% → 合併，保留最權威來源
- 同一 GitHub repo 的多個小更新 → 合併為一則
- RSS + Reddit + Web Search 報導同一事件 → 合併，加「多來源交叉報導」分

### 時事新聞去重
- 讀取昨日摘要，比對事件
- 同一事件只有「重大新進展」才納入，且聚焦新內容
- 同一事件被多國報導 → 不算重複，整合為 Format A（多元視角）

### 跨區塊去重
- 科技新聞中的重大事件（如 AI 政策）可能也出現在時事新聞
- 規則：時事版保留政治/政策角度，科技版保留技術角度
- 如果只有一個角度 → 只放在最合適的區塊
